# 提示詞 (Prompt) 參考範例

這裡為您整理出三個不同層次與詳細度的修正版本，您可以依據您對解答的需求來選擇最適合的一段：

### 版本一：詳細且結構化的提示詞（推薦👍，最能獲得完整教學）
> 「你現在是一位具備 ROS2、PX4 與 Gazebo 開發經驗的無人機數位孿生（Digital Twin）專家。我希望藉由使用開源專案（如 Titan Dynamics 的飛行器模型）來實踐從 CAD 到模擬環境的整合。請為我提供一份完整的 SOP（標準作業流程），包含：
> 1. **資料搜集**：如何在 Onshape、GitHub 或其他網站上，有效率地尋找並匯出合適的開源無人機 / 機器人 CAD 專案？
> 2. **模型轉換**：如何將這些 CAD 模型（包含質量、慣性矩與關節限制）轉換為 URDF 與 SDF 格式？（請介紹例如 `onshape-to-robot` 等自動化工具的使用手法）。
> 3. **模擬器整合**：如何將轉換好的 SDF/URDF 模型匯入 Gazebo，並順利套用 PX4 的物理設定（SITL）與串接 ROS2 節點（如 micro-ROS / XRCE-DDS）？」

---

### 版本二：精簡直接（直接切入重點操作）
> 「請提供一個完整的教學流程，說明如何從 Onshape 或 GitHub 尋找開源的飛行器專案（例如 Titan Dynamics），並將其 CAD 模型轉換成 URDF 與 SDF 格式。教學內容需要特別針對『如何將生成的模型匯入 Gazebo 模擬器』且能成功『搭配 ROS2 與 PX4 進行驗證』提供實作步驟及推薦的工具鏈。」

---

### 版本三：偏向求助與排除困難（適合後續遇到困難時發問）
> 「我在嘗試將 Titan Dynamics 這類的開源 CAD 專案用於 PX4 與 ROS2 的 Gazebo 模擬中。請問完整的『CAD to URDF/SDF』工作流程是什麼？在使用工具（例如 onshape-to-robot）轉換時，對於各機件間的關節（Joint）與旋轉軸設定，有沒有什麼常見的坑或是必須遵照的規範？」
